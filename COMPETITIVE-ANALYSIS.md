## 1. 핵심 포지셔닝

SENTINEL-ASM 은 **"권한이 검증된 대상만, 비파괴로, 모든 생태계를 동시에, AI가 고치고 재검증하는"** 폐루프 보안 플랫폼이다.
기존 도구들은 이 4가지를 **동시에** 충족하지 못한다.

## 3. 차별화 포인트 요약

### 🥇 SENTINEL-ASM 만이 갖는 것 (타사 전무)

| 항목 | 설명 |
|---|---|
| **권한 검증 게이트 + egress 주입** | 동의 범위가 곧 네트워크 발신 통제 자료구조 — 무권한 발신이 물리적으로 불가 |
| **soft-404 이중 게이트** | SPA 환경 오탐 0% (측정값) — 경쟁사 대비 질적 차별 |
| **AI 폐루프 재검증** | 발견 → AI 수정 → 동일 엔진 재점검 → 차분으로 해소 자동 확인 |
| **변조방지 감사 해시체인** | 모든 게이트 판정·실행을 SHA-256 체인으로 법적 증빙 수준 보존 |
| **PII 체크섬 검증** | 주민번호(가중합)·신용카드(Luhn)로 오탐 최소화한 실제 개인정보 탐지 |

### 🟡 SENTINEL-ASM 이 우세한 것

| 항목 | vs 주요 경쟁사 |
|---|---|
| 이메일/DNS 정밀도 | SPF RFC 7208 PermError·DMARC 정렬·DKIM sha1/t=y·MTA-STS·TLS-RPT·BIMI·DNSSEC — ZAP/Nuclei 없음 |
| 컴플라이언스 매핑 8개 표준 | ISMS-P·ISO27001·PCI-DSS·NIST-CSF·OWASP ASVS·OWASP Top10·GDPR/개인정보보호법·전자금융감독규정 |
| 프로젝트 폴더 다생태계 SBOM | npm·PyPI·Maven·Cargo·Ruby·PHP·.NET·Go·Gradle·Terraform·CI 11종 동시 — Snyk과 유사하나 비파괴·무료 |
| CT 로그 마이닝 | 워드리스트 + crt.sh 병합 → 인증서 이력 기반 그림자 자산 발굴 |

### 🔴 현재 없는 것 (로드맵)

| 항목 | 이유 | 계획 |
|---|---|---|
| 인증 세션 점검 | 자격증명 주입 필요 | 세션 쿠키/헤더 주입 어댑터 |
| SAST | 소스코드 접근 필요 | Semgrep API 어댑터 |
| 클라우드 IAM | 클라우드 자격증명 필요 | ExternalIntelProvider 어댑터 |
| 실시간 위협 인텔 | 유료 피드 | HIBP/VirusTotal 무료 티어 연동 |
